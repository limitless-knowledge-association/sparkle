Port Configuration Conflict

The configured port could not be used because it's already in use by another process. The daemon has started on an ephemeral (temporary) port instead.

What Happened?

You configured a fixed port in the Configuration settings, but when the daemon tried to restart, that port was already occupied by another process (possibly another Sparkle daemon or a different application).

Current ephemeral port: Loading...

How to Fix This:

Option 1: Stop the process using the configured port and restart the daemon:

npx sparkle browser

Option 2: Choose a different port number in the Configuration settings.

Option 3: Remove the fixed port configuration to always use ephemeral ports.
